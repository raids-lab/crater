## 개요

`metrics-server`는 Kubernetes 용 가볍고 확장 가능한 리소스 사용량 메트릭 수집기입니다. Crater 플랫폼에서는 다음을 위한 **실시간 CPU 및 메모리 사용량 데이터**를 제공합니다:

- Pod 및 Node 리소스 모니터링
- 작업 스케줄링 결정
- 프론트엔드 시각화 (예: 대시보드, 사용량 통계)

Metrics Server 는 `kubectl top`, 수평 Pod 자동 확장기 (HPA) 및 Crater 작업 관리 UI 와 같은 Kubernetes 구성 요소에 **필수적**입니다.

---

## 의존성

Kubernetes 클러스터가 완전히 작동하고 kubelet 을 통해 노드 구성 요소와 통신할 수 있는 상태가 되어야 **이후에** metrics 서버를 배포해야 합니다.

외부 의존성은 필요하지 않지만 다음을 확인하세요:

- kubelet 의 `--read-only-port` 또는 `--authentication-token-webhook` 설정이 활성화되어 있는지 확인하세요.
- 선택한 metrics-server 버전과 호환되는 Kubernetes 버전인지 확인하세요.

---

## Crater 관련 주의사항

- Crater 는 UI 에서의 **실시간 리소스 표시**에 `metrics-server`를 사용합니다.
- 작업 제출 및 리소스 예측 모듈은 또한 이러한 값을 읽어 스케줄링 정확도를 향상시킬 수 있습니다.
- Prometheus 와 보완 관계에 있지만, 짧은 수명의 Pod 에 대해 더 빠른 샘플링을 제공합니다.
