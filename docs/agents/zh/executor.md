# 执行者智能体

> 基于证据和用户意图执行写操作。

---

## 角色

执行者根据规划者的意图、探索者的证据以及用户的明确请求，决定并执行写操作（停止作业、重新提交、隔离节点等）。所有写操作均需用户确认。

---

## 操作规划

```
证据 + action_intent + user_message
  ↓
LLM 决定操作 (run_json)：
  [
    { "tool": "stop_job", "args": {"job_name": "..."}, "title": "停止作业",
      "reason": "GPU 利用率为 0，建议释放资源", "depends_on": [] },
    { "tool": "resubmit_job", "args": {"job_name": "..."}, "title": "重新提交",
      "reason": "...", "depends_on": ["action_0"] }
  ]
  ↓
按依赖顺序执行
  ↓
写工具返回 confirmation_required → 暂停等待用户审批
```

### 操作依赖

操作可以声明 `depends_on: [action_id]`：
- 仅当所有依赖项满足时才执行操作
- 如果某个依赖项失败，则跳过依赖它的操作
- 支持多步操作（例如：停止旧作业 → 使用新配置重新提交）

---

## 关键行为

- **未经用户明确意图不执行任何操作**：执行者仅执行用户原始请求所隐含的写操作
- **确认是强制性的**：所有写工具返回 `confirmation_required`；用户必须在前端进行审批
- **读写权限**：如果在执行操作前需要更多信息，也可以调用只读工具
