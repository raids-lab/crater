**플랫폼 설정**의 **모델 및 데이터셋 다운로드 할당량**에서 모든 사용자의 다운로드 Job 제한을 구성할 수 있습니다. 저장하기 전에 전체 설정과 영향 범위를 표시하고 확인을 요청합니다.

## 설정 항목

- **다운로드 할당량 활성화**: 제한을 검사할지 제어합니다. 비활성화 중에도 Job 의 예약, 성공 및 해제 상태는 기록됩니다.
- **최대 동시 작업 수**: 화이트리스트에 없는 사용자별로 `Pending` 또는 `Downloading` 상태인 작업 수를 제한합니다.
- **롤링 시간 창(시간)**: 현재 시각부터 거슬러 올라가는 기간이며 고정된 시각 구간이 아닙니다.
- **시간 창 내 성공 다운로드 한도**: 실제 완료 시각을 기준으로 계산하며 실행 중인 Job 은 슬롯을 예약합니다.
- **화이트리스트**: 동시 작업 및 롤링 시간 창 검사를 건너뜁니다. 플랫폼 관리자도 자동 면제되지 않으며 명시적으로 추가해야 합니다.

## 설정 변경 시 동작

할당량이 비활성화된 동안 또는 화이트리스트 사용자가 만든 Job 을 포함하여 실제 생성된 모든 Job 의 작업자와 수명 주기를 기록합니다. 따라서 할당량을 활성화하면 기존 대기 또는 다운로드 중 작업이 즉시 동시 작업 수에 포함되고, 사용자를 화이트리스트에서 제거하면 해당 사용자의 실행 중 작업도 계산됩니다. 검사를 건너뛴 동안 성공한 Job 도 완료 시각이 롤링 시간 창 안에 있으면 계산됩니다.

실패, 일시 중지, 삭제 또는 Job 제출 실패 시 예약을 해제합니다. 일시 중지는 로그를 저장하고 Job 을 삭제하며 이미 받은 파일은 유지합니다. 재개와 재시도는 새 Job 을 만들고 작업을 수행한 사용자의 할당량을 확인합니다.

완료된 공용 리소스를 재사용하거나 기존 공용 다운로드에 연결하면 Job 이 생성되지 않아 할당량을 사용하지 않습니다. 요청자 수는 수요만 나타내며 공용 디렉터리 권한을 변경하지 않습니다.
