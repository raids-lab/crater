사전 큐는 작업이 클러스터 스케줄링에 들어가기 전에 적용되는 승인 제어입니다. 관리자는 관리자 / 더보기 / 사전 큐에서 제출 스위치, 활성화 주기, 사용자 리소스 제한을 조정할 수 있습니다.

## 사전 큐 정책

- **backfill 제출 허용**: 사용자가 backfill 작업을 만들 수 있습니다. backfill 작업은 현재 유휴 리소스만 사용하며 잠금 신청은 할 수 없습니다.
- **사용자 리소스 제한 활성화**: 제출 및 활성화 시 대상 큐의 사용자 리소스 사용량을 검사합니다.
- **일반 작업 대기 허용 시간**: 일반 작업이 대기 상태에 들어간 뒤 현재 정책으로 처리되기까지의 허용 시간입니다.
- **활성화 스캔 간격**: 백그라운드 작업자가 사전 큐 작업을 스캔하는 간격입니다.
- **회차별 최대 활성화 수**: 한 번의 스캔에서 활성화할 수 있는 최대 작업 수입니다.

## 사용자 리소스 제한

각 제한은 하나의 큐에 연결되며 해당 큐의 작업에만 적용됩니다. GPU 큐에는 별도 제한을 두고, CPU 큐는 운영 정책에 따라 활성화하는 것을 권장합니다.

- **활성 상태**: 비활성화된 규칙은 보존되지만 검사에서는 제외됩니다.
- **후보 작업 수**: 한 사용자가 해당 큐에서 동시에 활성화 후보에 올릴 수 있는 작업 수를 제한합니다.
- **CPU / 메모리 상한**: 빈 값은 제한 없음을 의미합니다. 값을 설정하면 현재 사용량과 대기 중인 작업 요청량을 합산해 검사합니다.
- **가속기 상한**: 모델과 수량 쌍으로 설정합니다. 규칙에 없는 모델은 해당 규칙에서 제한하지 않습니다.

설정 후 새 작업은 먼저 사전 큐에 들어갑니다. 큐 리소스, 사용자 제한, 작업 유형 규칙을 모두 만족하면 백그라운드 작업자가 작업을 클러스터로 활성화합니다.
