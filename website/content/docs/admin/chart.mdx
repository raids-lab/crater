Crater 是一个专为 Kubernetes 设计的综合性 AI 开发平台，提供 GPU 资源管理、容器化开发环境和工作流编排功能。本文档详细列出了通过 Helm 部署 Crater 时的所有可配置项。

## 基础配置 (Global Values)

| 参数 | 类型 | 默认值 | 描述 |

### 现有模型 Logo 首次回填

Chart 1.1.0 默认启用元数据刷新并写入结果。第一次任务会选中状态为 `Ready` 且尚无刷新时间的现有模型和数据集下载记录，回填 Logo、README、来源统计等元数据；成功后写入刷新时间，后续只按 `staleAfter` 刷新过期记录，不会每 6 小时全量重刷。

若希望配置生效后立即回填、不等待下一次 Cron 调度，可以手动从 CronJob 创建一次 Job：

```bash
kubectl create job --from=cronjob/model-metadata-refresh \
  model-metadata-refresh-initial -n crater
kubectl logs -n crater -f job/model-metadata-refresh-initial
```

如果同名 Job 已存在，请更换 Job 名称。需要先验证网络和元数据结果时，可临时将 `modelMetadataRefresh.apply` 设为 `false` 后触发任务；此时只输出日志，不写数据库。

Logo 只会从 `backendConfig.modelMetadata.logoAllowedHosts` 中列出的精确 HTTPS 主机抓取，且每次重定向都会重新校验。使用镜像站或其他可信 CDN 时，应显式追加对应主机：

```yaml
backendConfig:
  modelMetadata:
    logoAllowedHosts:
      - huggingface.co
      - cdn-avatars.huggingface.co
      - resouces.modelscope.cn
      - resources.modelscope.cn
      - your-trusted-logo-cdn.example.com
```

该任务只处理已有的 `Ready` 下载记录；未完成或失败的下载不会被当作现有资源回填。
