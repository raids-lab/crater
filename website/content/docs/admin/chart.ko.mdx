### 기존 모델 Logo 최초 백필

Chart 1.1.0은 메타데이터 갱신과 데이터베이스 기록을 기본적으로 활성화합니다. 첫 실행에서는 상태가 `Ready`이고 아직 갱신 시간이 없는 기존 모델 및 데이터셋 다운로드 레코드를 선택하여 Logo, README, 소스 통계 등의 메타데이터를 백필합니다. 성공한 레코드에는 갱신 시간이 기록되므로 이후에는 6시간마다 전체를 다시 처리하지 않고 `staleAfter`보다 오래된 레코드만 갱신합니다.

다음 Cron 일정을 기다리지 않고 설정 적용 직후 백필하려면 CronJob에서 일회성 Job을 생성합니다.

```bash
kubectl create job --from=cronjob/model-metadata-refresh \
  model-metadata-refresh-initial -n crater
kubectl logs -n crater -f job/model-metadata-refresh-initial
```

같은 이름의 Job이 이미 있으면 다른 이름을 사용하십시오. 먼저 네트워크 연결과 메타데이터 결과를 검증하려면 Job 실행 전에 `modelMetadataRefresh.apply`를 일시적으로 `false`로 설정하십시오. dry-run 모드는 결과를 로그로만 출력하고 데이터베이스를 갱신하지 않습니다.

Logo는 `backendConfig.modelMetadata.logoAllowedHosts`에 정확히 등록된 HTTPS 호스트에서만 가져오며, 리디렉션의 각 단계에서도 다시 검증합니다. 미러 또는 다른 신뢰할 수 있는 CDN을 사용하는 경우 해당 호스트를 명시적으로 추가하십시오.

```yaml
backendConfig:
  modelMetadata:
    logoAllowedHosts:
      - huggingface.co
      - cdn-avatars.huggingface.co
      - your-trusted-logo-cdn.example.com
```

갱신 작업은 기존 `Ready` 다운로드 레코드만 처리합니다. 완료되지 않았거나 실패한 다운로드는 이 백필의 기존 리소스로 처리하지 않습니다.
