## モデルとは

モデルは共有ストレージ上の特定の場所を指す読み取り専用リソースで、マウントや共有に利用できます。Crater は ModelScope または HuggingFace からモデルを直接ダウンロードでき、ファイルシステムへアップロード済みのディレクトリをモデルとして登録することもできます。

## モデルをどこで確認するか

`データ管理-モデル`の下でモデルを確認できます。ここに表示されるモデルは、ユーザー自身が作成したモデル、個人に共有されたモデル、およびアカウントに共有されたモデルを含みます。

![model](./img/model.webp)

各モデルには基本情報と利用可能な操作が表示されます。モデル詳細の「ファイル」タブでは、実際の保存パスの確認とコピー、ファイルの閲覧ができます。過去のダウンロードではソースとリビジョンを含む長いパスが表示される場合があります。既存ジョブがそのパスを使用している可能性があるため、Crater はそれらのディレクトリを自動移動しません。

## リポジトリからモデルをダウンロードする

「モデルをダウンロード」を選択し、ソース、リポジトリ ID（例：`Qwen/Qwen3-32B`）、リビジョン、必要に応じてアクセストークンを入力します。新規ダウンロードの標準パスは次のとおりです。

```text
public/Models/<owner>/<repository>
```

同じリポジトリ ID については 1 つの公開リソースを使用します。Ready、保留中、ダウンロード中、または一時停止中の記録がすでにある場合、ソース、リビジョン、過去のパス形式が異なっても、既存の記録と実際のパスを再利用し、重複ダウンロードを行いません。申請した各ユーザーは 1 回だけ関連付けられ、記録の参照数は関連ユーザー数を表します。既存の記録が失敗している場合は、別のソースやリビジョンを要求する前に、その記録を再試行または処理してください。

ダウンロードに失敗した場合、同じ記録の再試行で継続できるよう、部分ファイルは元のディレクトリに残ります。完全に利用可能なのは Ready 状態の記録だけです。ダウンロードタスクを削除しても保存済みファイルは削除されません。失敗ディレクトリを消す場合は、再試行、マウント、利用者から参照されていないことを確認してから、管理者がクリーンアップしてください。

## モデルはどのように作成するか

モデルファイルがすでにストレージにある場合は「モデルを作成」を選択し、モデル名と説明を入力してフォルダを選び、必要に応じてタグと上流リポジトリのアドレスを追加します。

![model-create](./img/model-create.webp)

作成したモデル名は同じであってはいけません。フォルダを選択する際には、見える公共、個人、現在のアカウントのスペースファイルが自動的に表示され、そこから選択できます。

![alt text](./img/select-file.webp)

## モデルはどのように使用するか

新しい作業のページで、右側にはデータマウントボックスがあります。データマウントを追加した後、モデルを選択し、それをコンテナ内にマウントできます。

![alt text](./img/mount.webp)
