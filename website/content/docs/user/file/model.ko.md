## 모델이란 무엇인가

모델은 공유 스토리지의 특정 위치를 가리키는 읽기 전용 리소스로, 쉽게 마운트하고 공유할 수 있습니다. Crater는 ModelScope 또는 HuggingFace에서 모델을 직접 다운로드할 수 있으며, 파일 시스템에 이미 업로드된 디렉터리를 모델로 등록할 수도 있습니다.

## 모델을 어디에서 확인할 수 있나요

`데이터 관리 - 모델` 메뉴에서 모델을 확인할 수 있습니다. 여기에 표시되는 모델은 사용자가 직접 생성한 모델, 개인에게 공유된 모델, 계정에 공유된 모델을 포함합니다.

![model](./img/model.webp)

각 모델에는 기본 정보와 사용 가능한 작업이 표시됩니다. 모델 상세의 **파일** 탭에서 실제 저장 경로를 확인하고 복사하거나 파일을 탐색할 수 있습니다. 이전 다운로드는 소스와 리비전이 포함된 긴 경로를 사용할 수 있습니다. 기존 작업이 해당 경로에 의존할 수 있으므로 Crater는 이 디렉터리를 자동으로 이동하지 않습니다.

## 저장소에서 모델 다운로드

**모델 다운로드**를 선택한 다음 소스, 저장소 ID(예: `Qwen/Qwen3-32B`), 리비전, 선택적 액세스 토큰을 입력합니다. 새 다운로드는 다음 표준 경로를 사용합니다.

```text
public/Models/<owner>/<repository>
```

동일한 저장소 ID에는 하나의 공용 리소스만 사용합니다. Ready, 대기 중, 다운로드 중 또는 일시 중지된 기록이 이미 있으면 소스, 리비전 또는 이전 경로 형식이 달라도 기존 기록과 실제 경로를 재사용하며 중복 다운로드하지 않습니다. 요청한 사용자는 한 번만 연결되며 기록의 참조 수는 연결된 사용자 수를 의미합니다. 기존 기록이 실패한 경우 다른 소스나 리비전을 요청하기 전에 해당 기록을 다시 시도하거나 정리해야 합니다.

다운로드가 실패하면 같은 기록을 다시 시도할 때 이어받을 수 있도록 부분 파일이 원래 디렉터리에 남습니다. Ready 상태의 기록만 완전하고 사용 가능한 모델을 의미합니다. 다운로드 작업을 삭제해도 저장된 파일은 삭제되지 않습니다. 실패한 디렉터리를 정리하려면 재시도, 마운트 또는 다른 사용자가 해당 경로를 사용하지 않는지 확인한 후 관리자에게 요청하세요.

## 모델은 어떻게 생성하나요

모델 파일이 이미 스토리지에 있다면 **모델 만들기**를 선택하고 모델 이름과 설명을 입력한 뒤 폴더를 선택하고 필요에 따라 태그와 업스트림 저장소 주소를 추가합니다.

![model-create](./img/model-create.webp)

생성된 모델의 이름은 중복될 수 없으며, 폴더 선택 시 공개, 개인, 현재 계정의 공간 파일 중에서 볼 수 있는 폴더가 자동으로 나타나 선택할 수 있습니다.

![alt text](./img/select-file.webp)

## 모델은 어떻게 사용하나요

새로운 작업을 생성하는 페이지에서 오른쪽에 데이터 마운트 박스가 있으며, 데이터 마운트를 추가한 후 모델을 선택하여 컨테이너에 마운트할 수 있습니다.

![alt text](./img/mount.webp)
