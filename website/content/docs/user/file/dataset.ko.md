# 데이터셋

## 데이터셋이란 무엇인가

데이터셋은 공유 스토리지의 특정 위치를 가리키는 읽기 전용 리소스로, 쉽게 마운트하고 공유할 수 있습니다. Crater는 ModelScope 또는 HuggingFace에서 데이터셋을 직접 다운로드할 수 있으며, 파일 시스템에 이미 업로드된 디렉터리를 데이터셋으로 등록할 수도 있습니다.

## 데이터셋/모델과 공유 파일의 차이점

데이터셋/모델은 주로 읽기 전용 파일을 제공합니다. 이후 데이터셋/모델을 읽기 전용 공유 폴더로 이동하고, 데이터셋/모델에 대한 가속 기술을 적용하여 훈련 효율성을 높일 예정입니다. 공유 파일은 개인 파일을 타인과 공유하고, 타인이 읽고 쓸 수 있도록 허용합니다. 공용 공간의 파일에 대해 읽기 및 쓰기 권한이 필요할 경우 관리자에게 문의하시기 바랍니다.

## 데이터셋을 어디서 볼 수 있나요

`데이터 관리 - 데이터셋`에서 데이터셋을 볼 수 있습니다. 여기에 표시되는 데이터셋은 사용자가 직접 생성한 데이터셋, 개인에게 공유된 데이터셋, 계정에 공유된 데이터셋을 포함합니다.

![dataset](./img/dataset.webp)

각 데이터셋에는 기본 정보와 사용 가능한 작업이 표시됩니다. 데이터셋 상세의 **파일** 탭에서 실제 저장 경로를 확인하고 복사하거나 파일을 탐색할 수 있습니다. 이전 다운로드는 소스와 리비전이 포함된 긴 경로를 사용할 수 있습니다. 기존 작업이 해당 경로에 의존할 수 있으므로 Crater는 이 디렉터리를 자동으로 이동하지 않습니다.

## 저장소에서 데이터셋 다운로드

**데이터셋 다운로드**를 선택한 다음 소스, 저장소 ID, 리비전, 선택적 액세스 토큰을 입력합니다. 새 다운로드는 다음 표준 경로를 사용합니다.

```text
public/Datasets/<owner>/<repository>
```

동일한 저장소 ID에는 하나의 공용 리소스만 사용합니다. 기존 Ready, 대기 중, 다운로드 중 또는 일시 중지된 기록과 실제 경로를 재사용하고 요청한 사용자를 한 번만 연결합니다. 참조 수는 연결된 사용자 수를 의미합니다. 기존 기록이 실패한 경우 다른 소스나 리비전을 요청하기 전에 해당 기록을 다시 시도하거나 정리해야 합니다.

다운로드가 실패하면 같은 기록을 다시 시도할 때 이어받을 수 있도록 부분 파일이 원래 디렉터리에 남습니다. Ready 상태의 기록만 완전하고 사용 가능한 데이터셋을 의미합니다. 다운로드 작업을 삭제해도 저장된 파일은 삭제되지 않습니다. 실패한 디렉터리를 정리하려면 재시도, 마운트 또는 다른 사용자가 해당 경로를 사용하지 않는지 확인한 후 관리자에게 요청하세요.

## 데이터셋을 어떻게 생성하나요

데이터셋 페이지의 좌상단에 데이터셋 생성 버튼이 있으며, 클릭 후 이름, 설명을 입력하고 파일 폴더 위치를 선택하여 생성할 수 있습니다.

![dataset-create](./img/dataset-create.webp)

생성된 데이터셋 이름은 중복될 수 없으며, 폴더를 선택할 때는 공용, 개인, 현재 계정의 공간 파일 중에서 볼 수 있는 파일들을 자동으로 표시하여 선택할 수 있습니다.

![alt text](./img/select-file.webp)

## 데이터셋을 어떻게 사용하나요

새로운 작업 페이지의 오른쪽에는 데이터 마운트 창이 있으며, 데이터 마운트를 추가한 후 데이터셋을 선택하여 컨테이너에 마운트할 수 있습니다.
