## 할당량을 사용하는 작업

관리자는 사용자별로 동시에 대기 중 또는 다운로드 중일 수 있는 작업 수와 롤링 시간 창 안에서 성공할 수 있는 다운로드 수를 설정할 수 있습니다. Kubernetes 다운로드 Job 을 만들면 즉시 시간 창 슬롯 하나를 예약하여 이전 다운로드가 끝나기 전에 제한 없이 작업을 제출하지 못하게 합니다.

새 다운로드 생성, 실패 작업 재시도, 일시 중지 작업 재개는 모두 실제 작업을 수행한 사용자의 할당량을 확인합니다. 성공하면 실제 완료 시각부터 롤링 시간 창에 포함됩니다. 실패, 일시 중지, 삭제 또는 Job 제출 실패 시 예약을 해제하며 성공 다운로드 수에 포함하지 않습니다.

## 공용 리소스 재사용

같은 공용 모델 또는 데이터셋은 하나만 저장됩니다. 완료된 공용 리소스를 재사용하거나 이미 대기 중, 다운로드 중 또는 일시 중지된 공용 작업에 연결하면 새 Job 이 생성되지 않으므로 다운로드 할당량을 사용하지 않습니다. 다만 현재 사용자가 해당 리소스를 요청했다는 사실은 기록됩니다.

요청자 수는 수요만 기록하며 접근 권한이나 공유 범위를 변경하지 않습니다. 공용 저장소의 파일에는 기존 공용 디렉터리 접근 규칙이 그대로 적용됩니다.

## 일시 중지, 재시도 및 재개

일시 중지하면 최신 로그를 저장하고 실행 중인 Kubernetes Job 을 삭제하여 전송을 멈춥니다. 이미 기록된 파일은 유지됩니다. 재개하면 새 Job 을 만들고 가능한 경우 이어받으며 재개를 수행한 사용자의 할당량을 확인합니다.

실패 작업 재시도 역시 재시도한 사용자에게 계산됩니다. 다른 사용자의 해제된 시도는 현재 사용자의 할당량을 사용하지 않습니다. 할당량이 비활성화되어 있거나 사용자가 화이트리스트에 있으면 Job 을 차단하지 않지만, 나중에 할당량을 활성화하거나 사용자를 화이트리스트에서 제거할 때 실행 중 작업을 올바르게 계산할 수 있도록 수명 주기를 계속 기록합니다.
