Crater 플랫폼을 사용하려면 유효한 플랫폼 계정이 있어야 합니다. 플랫폼 설정에 따라 연구실의 LDAP 계정을 사용하여 로그인하거나 Crater 플랫폼 계정을 통해 액세스할 수 있습니다.

이 문서에 언급된 일부 로그인 방식의 입구가 보이지 않는다면 이는 정상입니다. 어떤 로그인 방식을 사용할 수 있는지는 플랫폼 관리자의 설정에 따라 다릅니다.

## LDAP 로그인 및 등록

### 개요
Crater는 LDAP 계정의 자동 등록 및 정보 동기화를 지원합니다. 계정을 수동으로 생성할 필요 없이 LDAP 계정 아이디와 비밀번호를 입력하기만 하면 로그인할 수 있습니다.

ACT 실험실 구성원이고 실험실 통합 인증 계정을 보유하고 있다면 이 방식을 사용해야 합니다.

### 인증 프로세스

<Mermaid
  chart="
flowchart TD;
    A[LDAP 아이디/비밀번호 입력] --> B{LDAP 서버 인증};
    B -- 인증 실패 --> C[자격 증명 오류 반환];
    B -- 인증 성공 --> D{Crater DB에<br/>사용자가 존재하는가?};
    D -- 아니요 --> E1[LDAP 속성 또는 외부 서비스에서<br/>사용자 UID/GID 가져오기];
    E1 --> E2[Crater DB 내에 계정을<br/>자동 생성하고 속성 동기화];
    D -- 예 --> F[이메일, 닉네임 등<br/>Crater DB 내 속성 업데이트];
    E2 --> G[기본 큐 권한 할당];
    F --> H[JWT 토큰 생성];
    G --> H;
    H --> I[로그인 성공];
"
/>

### 오류 빠른 확인

<Callout>
현재 플랫폼의 오류 메시지 처리 체계를 재구축 중입니다. 재구축 후 구체적인 오류 메시지 빠른 확인 문서를 보완할 예정입니다.
</Callout>

## 플랫폼 로그인

### 개요
Crater에서 수동으로 계정을 등록했거나 관리자가 계정을 할당하면서 이 로그인 방식을 사용하도록 요청한 경우 이 방식으로 로그인하십시오.

일반적으로 이 로그인 방식을 사용하여 LDAP 계정이나 다른 계정으로 직접 로그인할 수는 없다는 점에 유의하십시오.

### 인증 프로세스

<Mermaid
  chart="
flowchart TD;
    A[Crater 계정 아이디/비밀번호 입력] --> B{Crater DB 비교<br/>Bcrypt 검증};
    B -- 검증 실패 --> C[아이디 또는 비밀번호 오류 반환];
    B -- 검증 성공 --> D{사용자 상태 확인};
    D -- 비활성/중지 --> E[사용자 미활성화 반환];
    D -- 활성 --> F[관련 프로젝트/큐 가져오기];
    F --> G[JWT 토큰 생성];
    G --> H[로그인 성공];
"
/>

### 오류 빠른 확인

<Callout>
현재 플랫폼의 오류 메시지 처리 체계를 재구축 중입니다. 재구축 후 구체적인 오류 메시지 빠른 확인 문서를 보완할 예정입니다.
</Callout>

## 플랫폼 등록

### 개요
관리자가 '자율 등록 허용' 기능을 활성화한 경우 아직 계정이 없는 신규 사용자는 등록 페이지를 통해 Crater 플랫폼 계정을 생성할 수 있습니다.

### 인증 프로세스

<Mermaid
  chart="
flowchart TD;
    A[아이디 및 비밀번호 작성] --> B{플랫폼 등록 스위치 확인};
    B -- 꺼짐 --> C[자율 등록 금지 안내];
    B -- 켜짐 --> D{아이디 중복 확인};
    D -- 이미 존재 --> E[아이디 충돌 안내];
    D -- 존재하지 않음 --> F1[Crater 기본 UID/GID 1001 할당];
    F1 --> F2[Crater DB 내에 사용자 기록 생성];
    F2 --> G[사용자 공간 초기화];
    G --> H[기본 공용 큐 연결];
    H --> I[등록 성공 및 자동 로그인];
"
/>

### 오류 빠른 확인

<Callout>
현재 플랫폼의 오류 메시지 처리 체계를 재구축 중입니다. 재구축 후 구체적인 오류 메시지 빠른 확인 문서를 보완할 예정입니다.
</Callout>

## 자주 묻는 질문

### 1. ACT 실험실 사용자의 경우 실험실 외부에서 GPU 클러스터에 어떻게 접속합니까?

클라이언트 다운로드: https://vpn.buaa.edu.cn/portal,
로그인 창 하단의《클라이언트 다운로드》를 클릭하고 팝업 창에서 해당 운영체제를 선택하여 다운로드합니다.

### 2. ACT VPN에 연결할 수 없는 경우의 대처 방법:

로컬 DNS 해석이 올바른지 확인하십시오;

```
dig vpn.act.buaa.edu.cn
```

결과가 없는 경우 `219.224.171.1`을 `vpn.act.buaa.edu.cn` 대신 사용하여 접속할 수 있습니다.
